Echter Lauf, 3 Quellen · kein Happy-Path-Denken: Dropout an jeder Stufe ist normales B2B-Scraping. ImmoScout24 geblockt, Immowelt ohne Firmennamen — reale Ausfallquoten.

01
Scraping — 3 Portale
Kleinanzeigen.de, Immowelt.de, ImmoScout24.de. Nur gewerbliche Anbieter — Privat-Verkäufer werden sofort gefiltert. Kleinanzeigen liefert den Firmennamen erst auf der Detailseite, die parallel abgerufen wird (5 Threads).
scraper.py → scrape_kleinanzeigen() / scrape_immowelt() / scrape_immoscout()
02
Deduplication
Hash über Firmenname und Listing-URL. Kein Lead erscheint doppelt, auch wenn er auf mehreren Portalen inseriert. Leads ohne Firmenname werden für das Enrichment markiert.
main.py → deduplicate(leads)
03
Website finden
DuckDuckGo-Suche nach Firmenname + Stadt + Impressum. Domain-Guess als Fallback – konstruiert mögliche .de-Domains direkt aus dem Firmennamen und prüft sie. Portale und Social Media werden gefiltert.
enricher.py → find_company_website()
04
Impressum laden — der eigentliche Key Insight
12 Standard-Pfade werden geprüft (/impressum, /kontakt, /about usw.). §5 TMG verpflichtet jede deutsche Firma gesetzlich zum Impressum – Name, Adresse, E-Mail müssen veröffentlicht sein. Damit ist das Impressum die zuverlässigste strukturierte B2B-Kontaktquelle im deutschsprachigen Markt – kein Scraping-Graubereich, keine Login-Wall, keine Rate-Limits.
enricher.py → fetch_impressum(base_url)
05
Kontakt extrahieren
Zwei Modi, explizit getrennt:
MIT API-KEY
Claude Haiku / GPT-4o liest Anrede, Nachname und E-Mail strukturiert aus dem Impressum-Rohtext → alle Copy-Variablen befüllt → copy-ready (Vorname wird extrahiert wenn vorhanden, ist aber kein Pflichtfeld)
OHNE API-KEY (Regex)
Regex findet E-Mail-Adressen zuverlässig → Namen unzuverlässig (Impressum-Format variiert) → System degradiert statt zu crashen
enricher.py → extract_contact_llm() + _extract_contact_regex()
06
Export
CSV kampagnenfähig mit allen Copy-Variablen — direkt importierbar in Outreach-Tools. Geplante Erweiterungen:
Geplant: Close.io Write-Back → Tag shx_enriched Geplant: Webhook POST an n8n / Zapier / Make
main.py → export_csv()

from models  import Lead
from enricher import enrich_lead

app = FastAPI()

@app.post("/enrich", response_model=Lead)
def enrich(company_name: str, city: str = ""):

Bug models.py #01
Das System lehnte fertige Leads ohne Grund ab
Problem

Das System prüfte, ob ein Vorname vorhanden ist — obwohl die E-Mail-Vorlage gar keinen Vornamen enthält. Fast alle fertig angereicherten Leads wurden dadurch als „nicht versandbereit“ abgelehnt, obwohl Anrede, Nachname, E-Mail-Adresse und Stadt vollständig vorlagen.

Fix
self.Anrede and self.firstName and self.lastName and self.email and self.Stadt self.Anrede and self.lastName and self.email and self.Stadt
↑ Copy-ready Rate maximiert
Bug scraper.py #02
Der Scraper blätterte nie auf die nächste Seite weiter
Problem

Kleinanzeigen.de hatte die Adressstruktur seiner Unterseiten geändert. Das Tool versuchte mit dem alten Format weiterzuzblättern — die Website ignorierte das einfach und lieferte immer wieder dieselbe erste Seite. Statt drei verschiedene Seiten zu sammeln, wurde dieselbe dreimal geladen.

Fix
/s-immobilien-kaufen-neubau/?page=2 /s-immobilien-kaufen-neubau/seite:2/k0
↑ Scraping-Volumen verdreifacht
Bug enricher.py #03
Die Hälfte aller KI-Antworten wurde still verworfen
Problem

Das KI-Modell liefert seine strukturierten Kontaktdaten manchmal in zwei leicht unterschiedlichen Formaten zurück. Das Tool erkannte nur eines davon — beim anderen wurden die Antwort kommentarlos verworfen und das System wechselte in den deutlich schlechteren Notfall-Modus, ohne dabei irgendeine Fehlermeldung zu zeigen.

Fix
re.sub(r"^```json", "", text) re.sub(r"^```(?:json)?", "", text)
↑ KI-Extraktion deutlich zuverlässiger
Bug enricher.py #04
Das Tool startete gar nicht ohne eingetragenen API-Schlüssel
Problem

Beim Start versuchte das Tool sofort eine Verbindung zur KI-Schnittstelle herzustellen — egal ob ein API-Schlüssel eingetragen war oder nicht. Ohne Schlüssel stürzte das System sofort ab, anstatt einfach in den eingebauten Notfall-Modus zu wechseln, der auch ohne KI funktioniert.

Fix
anthropic_client = anthropic.Anthropic() anthropic_client = ( anthropic.Anthropic() if os.getenv("ANTHROPIC_API_KEY") else None )
↑ Läuft jetzt auch ohne API-Schlüssel stabil
Verbesserung enricher.py #05
Impressum-Suche: 2 Minuten blindes Raten → in Sekunden gefunden
Problem

Das Tool hat bei jeder Firma einfach 8 typische Impressum-Adressen nacheinander ausprobiert — egal ob sie existieren oder nicht. Im schlimmsten Fall dauerte das über 2 Minuten pro Firma. Dazu kam: Firmen, die ihr Impressum an einer ungewöhnlichen Stelle abgelegt hatten (z. B. unter „Unternehmen“ statt direkt als „Impressum“), wurden komplett übersprungen.

Fix

Das Tool besucht jetzt zuerst die Startseite der Firma und liest alle Links durch — genau wie ein Mensch es tun würde. Sobald ein Link mit dem Wort „Impressum“ entdeckt wird, springt das Tool direkt dorthin. Nur wenn das nicht funktioniert, werden die alten Standard-Adressen als Backup durchprobiert.

↑ Worst Case 120 s → ~20 s · findet jetzt auch versteckte Impressen
